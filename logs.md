# Текущие продвижения.
## Идеал.
Мы хотим решить задачу эффективного поиска видео по описанию, то есть на вход мы получаем какое-то описание искомого видео, из этого описания мы вычленяем так много информации, как только можем, и по этой информации находим наиболее подходящие видео. Мотивировка этого проекта заключалась в том, что это интересная задача для построения смысловых эмбеддингов промптов, видео, поиска интересной метрики в пространстве этих эмбеддингов, возможного дообучения моделей для наших целей. С точки зрения общества это может быть использовано во многих сферах, начиная от рекомендательных систем, заканчивая какими-то системами безопасноти и защиты от нежелательного контента.

## Начало.
- Описание:
На этом этапе мы решаем достаточно упрощённую версию того, что хотели бы уметь решать в идеале: у нас есть размеченный по 8 эмоциями [корпус](https://arxiv.org/abs/2511.11002v1) видео, нам задаётся эмоциональный промпт, из которого мы извлекаем эмоцию, а затем производим поиск в нашем корпусе. 
- Метрики:
Так как сейчас всё достаточно просто, то и метрики у нас классические: 
    - Accuracy
    - Precision
    - Recall
    - F1
- Baseline:
В качестве бейзлайна выступает какая-то классическая моделька, например, SVM, с помощью которой мы классифицируем по эмоциями промпт. Можно ещё попробовать FastText, так как это всё ещё линейная модель, но при этом она позволяет использовать слова, на которых она напрямую не обучалась и делает это эффективно.
- Основная модель:
Сейчас в качестве основной модели для вычлинения эмоций используется Qwen, затем уже выдаётся результат по его метке.
